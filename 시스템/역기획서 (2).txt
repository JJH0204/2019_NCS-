중요성
포트폴리오, 신입은 게임 라이브에서 배치됨, 실무에 쓰는 일이 많음


잘쓰는 법
사용자 경험으로 전달되는 과정을 파악
단준정보의 나열이 아닌 실제 기획 사항에 초점 
기획서를 쓰는 순서로 역기획서 작성


작성 순서
1. 게임 선정
1. 어떤 게임을 만들 것지 확인 ( 게임 선정 )
   1. 장르 나열
   2. 메커니즘 분석
   3. 선택
2. 어떤 회사에 지원 할 것인지? ( 입사 지원서, 포폴용 )
   1. 지원할 회사의 게임을 분석
   2. 게임 선택시 유의 사항
      1. 서비스 기간 6개월 이하 제외
      2. 서비스 기간 3년 이상 제외
      3. 되도록 상업적으로 성공한 게임 ( 순위권 매출 순위권 )
2. 게임 플레이
1. 일단 플레이 (즐기고 있다면 … )
2. 범위를 정해야 함 ( 레벨 디자인 : 맵구조, 유저 커뮤니티 : 파티/길드 )
3. 객관적 분석
1. 객관성 유지 ( 순위, 이벤트, 업데이트, 게임기사 등 토대로 )
4. 게임 진행 분석
1. 씬나누기 
2. 게임 진행 분석 설명
3. 자료 정리 ( 기사 자료, 인터뷰, 순위 자료 조사 정리
5. 전체 분석 순서
1. 관찰
2. 추론
3. 검증
4. 분석


역기획서 종류
1. 컨텐츠 역 기획서
: 컨텐츠 시스템이 구성된 이유 기획의도를 분석의 통해 유추
2. 시스템 역 기획서
: 구연하기 위해 어떤 방식으로 구성되어 있는지 파악
 게임 플레이 스크린샷 - > 게임 시스템 ( 알고리즘 및 플로우 차트 ) -> 게임 메커니즘 (메커니즘 정리 및 테이블)
3. ux / ui 역 기획서
: 사용자 경험, 그리고 편의성을 기준으로 왜 이렇게 구성되고 디자인 되었는지 파악
핵심 분석 항목
pc / 모바일 분석
크로스플렛폼 : 클라우드 플렛폼


수치적인 부분이 많은 경우 엑셀, 시스템 워드 ,  내용에 따라 ppt
정해진 문서 양식이 없음

3. 내가 담당 기획자라고 생각하고 작성 ( 효율을 생각 )
